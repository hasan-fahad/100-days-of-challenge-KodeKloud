# Day 003: Secure SSH Root Access

Disabling direct SSH root access is a critical security practice to protect the App Servers within the Stratos Datacenter.

---

## üõ†Ô∏è Objective

Disable **direct SSH root login** on all App Servers to enhance system security.

---

## ‚úÖ Steps

### 1. Login to the App Server
```bash
ssh user@app-server-ip
# or
ssh user@server-name
```

---

### 2. Edit SSH Configuration File
Open the SSH configuration file:

```bash
sudo vi /etc/ssh/sshd_config
```

Inside **vi editor**:

1. Press `i` ‚Üí *Insert mode*  
2. Modify the line:

   **From:**
   ```
   PermitRootLogin yes
   ```

   **To:**
   ```
   PermitRootLogin no
   ```
3. Press `Esc`  
4. Type:
   ```
   :wq
   ```
   to save & exit.

---

### 3. Restart SSH Service
```bash
sudo systemctl restart sshd
```

---

### 4. Verify Root Login Status
```bash
sudo cat /etc/ssh/sshd_config | grep PermitRootLogin
```

---

## üí° Good to Know

### **SSH Security Best Practices**
- Disable direct root login for better security  
- Use `sudo` for administrative privileges  
- Prefer **SSH key-based authentication**  
- Consider changing the default SSH port (22)

---

## üîß Useful SSH Configuration Options

| Option | Description |
|--------|-------------|
| `PermitRootLogin no` | Disable direct root login |
| `PasswordAuthentication no` | Disable password login |
| `PubkeyAuthentication yes` | Enable SSH key authentication |
| `Port 2222` | Change SSH port |

---

## üîê Security Benefits
- **Better Audit Trails:** `sudo` logs every action  
- **Least Privilege:** Users only get needed access  
- **Reduced Attack Surface:** Prevent brute force on root  
- **Accountability:** Individual user responsibility
