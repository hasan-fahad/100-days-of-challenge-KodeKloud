# Day 004: Script Execute Permissions

This guide walks you through granting executable permissions to a script on Linux.

---

## üéØ Objective

Grant executable permissions to `/tmp/xfusioncorp.sh` on App Server 3 and ensure all users can execute it.

---

## üîß Steps

### 1. Connect to App Server 3

```bash
ssh user@app-server-3-ip
```

### 2. Check Current Permissions

```bash
ls -la /tmp
```

Expected output:

```
---------- 1 root root   40 Dec  7 11:15 xfusioncorp.sh
```

### 3. Update Permissions

```bash
chmod 755 /tmp/xfusioncorp.sh
```

Explanation:

* 7 (rwx) ‚Üí User can read, write, execute
* 5 (r-x) ‚Üí Group can read, execute
* 5 (r-x) ‚Üí Others can read, execute

### 4. Verify Changes

```bash
ls -la /tmp
```

Expected output:

```
-rwxr-xr-x 1 root root   40 Dec  7 11:15 xfusioncorp.sh
```

## ‚öôÔ∏è Understanding chmod in Linux

### **Permissions Basics**

* **Read (r)** = 4
* **Write (w)** = 2
* **Execute (x)** = 1

### **Categories of Users**

* **User (u)** ‚Äì owner
* **Group (g)** ‚Äì users in same group
* **Others (o)** ‚Äì everyone else

### **Two Ways to Use chmod**

#### 1. Symbolic Mode

```bash
chmod u=rwx,g=rx,o=r test.sh
```

* `+` adds permission, `-` removes, `=` sets exact permission
  Example:

```bash
chmod g+w test.sh     # Add write for group
chmod o-r test.sh     # Remove read for others
```

#### 2. Numeric (Octal) Mode

* r = 4, w = 2, x = 1
* Sum per category for 3-digit code

```bash
chmod 754 test.sh
```

Breakdown:

* User: 7 = rwx
* Group: 5 = r-x
* Others: 4 = r--

### **Common Patterns**

| Mode | Permissions | Description         |
| ---- | ----------- | ------------------- |
| 755  | rwxr-xr-x   | Standard executable |
| 644  | rw-r--r--   | Standard file       |
| 600  | rw-------   | Private file        |
| 777  | rwxrwxrwx   | Full access (avoid) |

### **Security Considerations**

* Principle of Least Privilege
* Execute bit required for scripts
* Directory permissions must allow execute to access contents
* Sticky bit useful for shared directories (/tmp)

---

## ‚úÖ Best Practices

* Grant **minimum required permissions**
* Avoid `777` unless absolutely necessary
* Scripts used by multiple users should be readable and executable, not writable by all
